**Network for Orchestrated Virtual Agents.** Once you have more than one agent
working on something, you've stopped having an AI problem and started having a
distributed systems problem.

Two agents grab the same task. One dies mid-job holding a lock. Two write the
same value and one of them silently wins. None of this is new. Databases and
schedulers have been dealing with it for forty years, and the answers are
well known and pretty unglamorous, which might be why the current excitement
keeps rediscovering them the hard way.

NOVA gives agents those primitives directly: task distribution with claiming,
lease-based resource locking that survives a crash, message passing, and shared
state with optimistic concurrency control. It runs over QUIC, so one connection
carries many multiplexed streams and encryption comes for free.
